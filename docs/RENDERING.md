# Puppt Rendering

`puppt render` is implemented as a Puppt-owned `.pptx` interpretation and PNG output path. It must not shell out to LibreOffice, PowerPoint, Keynote, browser renderers, or image conversion tools.

The Apple Notes PNGs checked in under `testdata/realworld-ppts/reference-renders/manual-using-apple-note` are the current golden test targets for the persistent renderer goal. They are not part of the renderer implementation and are not invoked by the CLI. LibreOffice exports under `testdata/realworld-ppts/reference-renders/libreoffice-7.6.0.3` are retained as an alternate comparison set because different host renderers can differ visibly.

Current scope:

- `puppt render <input.pptx> --slide N --out slide-N.png --json`
- `--dpi` controls PNG output resolution; the default is 72 DPI, matching the 960x540 checked-in Apple Notes reference dimensions for the checked-in widescreen fixtures, while `--dpi 96` emits 1280x720 for alternate comparison sets
- `PUPPT_FONT_MAP` can pin exact font files with semicolon-separated entries such as `Calibri=/fonts/Calibri.ttf;Calibri:bold=/fonts/Calibri-Bold.ttf`; style keys support `bold`, `italic`, and `bolditalic`
- slide dimensions from `ppt/presentation.xml`
- deterministic PNG output via Go's `image/png`; the default 72 DPI emits 960x540 for 13.333x7.5 widescreen decks, while `--dpi 96` emits 1280x720
- final PNG colors are converted from DrawingML/source sRGB into Display P3 RGB using standard D65 sRGB and Display P3 transfer/matrix math, matching the checked-in Apple Notes reference render color pipeline while keeping Apple Notes out of Puppt's renderer
- solid and first-pass gradient slide backgrounds where explicitly present on the slide, layout, or master, including radial interpolation for circular gradients with `fillToRect` focus rectangles
- embedded PNG, JPEG, GIF, and simple SVG pictures with basic offset/extent placement, source cropping including negative `srcRect` padding, quarter-turn image rotation, PNG ICC matrix/TRC and Adobe RGB (1998) JPEG source color conversion, alpha compositing, deterministic interpolated resampling selected from the current checked-in comparison evidence, DrawingML outer shadows for supported picture geometry, Gaussian alpha-mask soft edges for rectangular pictures, and custom-geometry masks for single DrawingML paths made of move/line/cubic/close commands; SVG support covers Office icon-style `path`, `rect`, `circle`, and `ellipse` fills, while Office SVG blips preserve their raster fallback when one is available
- solid fills for rectangle, rounded rectangle, triangle, ellipse, right-arrow, and simple custom/freeform polygon shape geometry, with first-pass antialiasing for polygon edges
- outlines for rectangle, ellipse, triangle, right-arrow, chevron, notched-right-arrow, curved-arrow, and simple custom/freeform polygon geometry
- chevron and notched-right-arrow preset geometry use DrawingML adjustment guides for their polygon paths
- inherited master/layout render parts are traversed before slide objects, with inherited placeholders skipped as standalone painted objects
- master/layout background references through `p:bgRef` resolve theme `bgFillStyleLst` entries with `phClr` substituted from the reference color
- slide layouts with `showMasterSp="0"` keep master data available for inheritance but suppress master shapes during the paint pass
- slide placeholder text can inherit missing text-box properties from matching layout/master placeholders by placeholder type or index, can inherit transform bounds when no local bounds exist, and applies inherited default bullets for body/content placeholders while preserving explicit no-bullet paragraphs
- master `p:txStyles` title and center-title defaults are used as fallback font size, color, boldness, and alignment when slide text does not provide those properties locally
- DrawingML theme color schemes are read from package theme parts for common scheme color slots, with master and override color-map aliasing applied per rendered part and hardcoded Office fallback colors when a deck has no parseable theme
- common DrawingML color modifiers such as luminance/tint/shade/alpha are applied with bounded channel approximations, including combined luminance modifier/offset handling for common Office theme colors
- shape `p:style` fill and line references are used as a bounded fallback when direct shape properties do not provide colors
- shape outer shadows are parsed from `blurRad`, `dist`, `dir`, and color/alpha and painted through a separable Gaussian alpha mask for supported filled shape geometry; unsupported shadow geometry is reported explicitly
- shape text is rasterized with fonts resolved through the bounded `golang.org/x/image` primitive dependency documented in `docs/decisions/0003-use-go-x-image-for-rendering.md`, including exact local font use when available, exact configured font files from `PUPPT_FONT_MAP`, locally installed Carlito metric-compatible fonts before the bundled Carlito fallback for unavailable Calibri/Calibri Light theme fonts, configured local symbol-font substitutes for Segoe UI symbol fonts where available, unhinted font rasterization selected from the current checked-in comparison evidence, shape-level italic face selection, paragraph/run-level font family, font size, and bold face selection, authored bullet font families, theme bullet font references, DrawingML bullet text-font fallback, run-scoped text colors, run-level baseline offsets, explicit reporting for unresolved generic font fallback, and first-pass custom drawing/mapping for common Office symbol bullets
- grouped shapes and pictures are flattened with basic group coordinate transforms
- rectangle and ellipse outlines plus simple line, straight-connector, and `p:cxnSp` connector geometry are painted, including horizontal or vertical connectors with zero-height or zero-width extents
- triangle connector arrowheads are painted with DrawingML marker width/length sizing; non-triangle line-end markers are still reported as partial
- graphic-frame tables are painted with a first-pass grid layout that preserves authored row/column extents when they fit the graphic frame and otherwise scales to the frame, including direct cell fills/borders, parsed `ppt/tableStyles.xml` fills/borders/text styling for common table style regions, row spans/vertical merges, cell text, and straight border join metadata; unsupported table features such as non-solid cell fills, visible effects, non-flat line caps, compound lines, and line-end decorations are reported specifically
- diagram graphic frames can resolve related DrawingML diagram drawing parts, apply the package theme to their shape styles, paint their simple shapes/text inside the frame bounds, and report unsupported non-shape diagram content specifically
- text rendering preserves paragraph breaks, applies body insets including DrawingML defaults, supports simple left/center/right alignment, wraps words inside text bounds, expands unrotated text shapes with `a:spAutoFit` and widens no-wrap autofit boxes to contain measured text, applies explicit normal-autofit font and line-spacing reductions when present, derives bounded normal-autofit scaling when `a:normAutofit` is requested while allowing regular wrapping within explicit hard-break segments, honors local `a:bodyPr` text anchoring over placeholder inherited anchors, honors `spcFirstLastPara` for text-box anchoring, and uses first-pass DrawingML bullet prefixes and auto-numbered bullets plus inherited paragraph margins, hanging indents, line spacing, fixed-point and percent before/after spacing, bullet size, and paragraph `defRPr` font-family/font-size/color/bold defaults for body/content placeholders; unsupported text-body modes such as vertical text, body-level text rotation, multi-column text, and anchor-center are reported specifically instead of using the former blanket simplified-layout report
- explicit `unsupported` JSON entries for visible objects that are detected but not yet painted
- empty placeholder shells are ignored when they have no visible content to paint

Full parity remains in progress. The real-world comparison harness is gated behind `PUPPT_RUN_REALWORLD_RENDER_TESTS=1` and compares all 61 fixture slides against the checked-in Apple Notes reference PNGs by default. Set `PUPPT_REALWORLD_REFERENCE_ROOT` to compare against a different checked-in reference set such as the LibreOffice exports, and set `PUPPT_REALWORLD_ARTIFACT_DIR` to keep `got.png`, `reference.png`, and `diff.png` artifacts for failing slides.
