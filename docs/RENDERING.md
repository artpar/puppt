# Puppt Rendering

`puppt render` is implemented as a Puppt-owned `.pptx` interpretation and PNG output path. It must not shell out to LibreOffice, PowerPoint, Keynote, browser renderers, or image conversion tools.

The Apple Notes PNGs checked in under `testdata/realworld-ppts/reference-renders/manual-using-apple-note` are the current golden test targets for the persistent renderer goal. They are reference artifacts only: they are not part of the renderer implementation and are not invoked by the CLI. LibreOffice exports under `testdata/realworld-ppts/reference-renders/libreoffice-7.6.0.3` are retained only as a secondary comparison reference because different host renderers can differ visibly; they are not a renderer dependency or acceptance target.

M12 status on 2026-06-02: the renderer is not yet final-conformance complete.
Coverage, unit, CLI/JSON, preservation/reporting, and dependency-boundary
checks pass, but the exact Apple Notes gate still reports 61/61 differing
slides with 9,305,437 total differing pixels, and the tracked clean
micro-fixture suite still reports 59 failures in expected-failure accounting
mode. The M12 coverage matrix has 0 rows marked `Unimplemented / no evidence`;
remaining supported-scope visual gaps stay Partial until source-backed and
fixture-proven. Unsupported remains reserved for source-proven static-renderer
impossibilities with preservation/reporting; out-of-scope content is tracked
separately and cannot cover visible static PresentationML/DrawingML failures.

Current scope:

- `puppt render <input.pptx> --slide N --out slide-N.png --json`
- `--dpi` controls PNG output resolution; the default is 72 DPI, matching the 960x540 checked-in Apple Notes reference dimensions for the checked-in widescreen fixtures, while `--dpi 96` emits 1280x720 for alternate comparison sets
- `PUPPT_FONT_MAP` can pin exact font files with semicolon-separated entries such as `Calibri=/fonts/Calibri.ttf;Calibri:bold=/fonts/Calibri-Bold.ttf`; style keys support `bold`, `italic`, and `bolditalic`
- slide dimensions from `ppt/presentation.xml`
- deterministic PNG output via Go's `image/png`; the default 72 DPI emits 960x540 for 13.333x7.5 widescreen decks, while `--dpi 96` emits 1280x720
- final PNG colors are converted from DrawingML/source sRGB into Display P3 RGB using standard D65 sRGB and Display P3 transfer/matrix math and tagged with Display P3/sRGB `cICP` metadata, matching the checked-in Apple Notes reference render color pipeline while keeping Apple Notes out of Puppt's renderer
- solid and first-pass gradient slide backgrounds where explicitly present on the slide, layout, or master, including rectangular path gradients and source-backed radial interpolation for circular gradients with `fillToRect` focus ellipses; non-empty `tileRect`, non-`none` `flip`, and unsupported path variants are still reported explicitly
- embedded PNG, JPEG, GIF, and simple SVG pictures with basic offset/extent placement, source cropping including negative `srcRect` padding, DrawingML image rotation, PNG ICC matrix/TRC and Adobe RGB (1998) JPEG source color conversion, alpha compositing, deterministic interpolated resampling selected from the current checked-in comparison evidence, DrawingML outer shadows for supported picture geometry, Gaussian alpha-mask soft edges for rectangular pictures, and custom-geometry masks for single DrawingML paths made of move/line/cubic/close commands; SVG support covers Office icon-style `path`, `rect`, `circle`, and `ellipse` fills, including direct presentation fills plus simple class and inline CSS fill styles, while Office SVG blips preserve their raster fallback when one is available
- solid fills and gradient fills for rectangle, rounded rectangle, triangle, ellipse, right-arrow, and simple custom/freeform polygon shape geometry, with first-pass antialiasing for polygon edges, path-bounds-based gradient coordinates where shape paths are available, and supported circular path-gradient rendering using the DrawingML radial focus ellipse model
- outlines for rectangle, ellipse, triangle, right-arrow, chevron, notched-right-arrow, curved-arrow, and simple custom/freeform polygon geometry
- chevron and notched-right-arrow preset geometry use DrawingML adjustment guides for their polygon paths
- inherited master/layout render parts are traversed before slide objects, with inherited placeholders skipped as standalone painted objects
- master/layout background references through `p:bgRef` resolve theme `bgFillStyleLst` entries with `phClr` substituted from the reference color
- slide layouts with `showMasterSp="0"` keep master data available for inheritance but suppress master shapes during the paint pass
- slide placeholder text can inherit missing text-box properties from matching layout/master placeholders by placeholder type or index, can inherit transform bounds when no local bounds exist, and applies inherited default bullets for body/content placeholders while preserving explicit no-bullet paragraphs
- master `p:txStyles` title and center-title defaults are used as fallback font size, color, boldness, and alignment when slide text does not provide those properties locally; master `otherStyle` paragraph defaults are also applied to non-placeholder shape text without overriding local paragraph properties
- DrawingML theme color schemes are read from package theme parts for common scheme color slots, with master and override color-map aliasing applied per rendered part and hardcoded Office fallback colors when a deck has no parseable theme
- common DrawingML color models and modifiers are resolved for supported paint paths, including sRGB, linear-gamma scRGB converted through the sRGB transfer function, luminance/tint/shade/alpha modifiers, and combined luminance modifier/offset handling for common Office theme colors
- shape `p:style` fill and line references are used as a bounded fallback when direct shape properties do not provide colors
- shape outer shadows are parsed from `blurRad`, `dist`, `dir`, `algn`, `rotWithShape`, `sx`, `sy`, `kx`, `ky`, and color/alpha and painted through a separable Gaussian alpha mask for supported filled shape geometry; unsupported shadow geometry and unsupported shadow scale/skew or rotate-with-shape transforms are reported explicitly
- shape text is rasterized with fonts resolved through the bounded `golang.org/x/image` primitive dependency documented in `docs/decisions/0003-use-go-x-image-for-rendering.md`, including exact local font use when available, exact configured font files from `PUPPT_FONT_MAP`, Microsoft Office cloud-font cache discovery, Microsoft font folders, Linux user/system font roots, and canonical Office Calibri filenames for local Calibri/Calibri Light installs, locally installed Carlito metric-compatible fonts before the bundled Carlito fallback for unavailable Calibri/Calibri Light theme fonts, partial reporting when Calibri/Calibri Light substitution is used, exact local or configured Segoe UI Symbol/Historic fonts when available with generic fallback reporting when those proprietary Microsoft fonts are missing, unhinted font rasterization selected from the current checked-in comparison evidence, shape-level italic face selection, paragraph/run-level font family, `a:sym` symbol typefaces for private-use symbol runs, font size, bold face selection, authored character spacing, authored bullet font families, theme bullet font references, DrawingML bullet text-font fallback, run-scoped text colors, run-level baseline offsets, attribute and explicit `uLn` underlines with solid `uFill` underline color, explicit reporting for unresolved generic font fallback, and first-pass custom drawing/mapping for common Office symbol bullets
- grouped shapes and pictures are flattened with basic group coordinate transforms
- picture and shape image fills honor DrawingML crop, flip, concrete black/white modes (`gray`, `black`, `white`, and `hidden`), alpha modulation, custom masks, soft edges, `CT_Blip@cstate` compression metadata in debug summaries, and `blipFill@rotWithShape` for deciding whether raster pixels rotate with the shape; other `ST_BlackWhiteMode` values are reported explicitly until implemented
- rectangle and ellipse outlines plus simple line, straight-connector, and `p:cxnSp` connector geometry are painted, including horizontal or vertical connectors with zero-height or zero-width extents and antialiased preset/custom dashed strokes for default square-cap line rendering
- triangle connector arrowheads are painted with DrawingML marker width/length sizing; non-triangle line-end markers are still reported as partial
- graphic-frame tables are painted with a first-pass grid layout that preserves authored row/column extents when they fit the graphic frame and otherwise scales to the frame, derives row proportions from measured source text heights when all authored row heights are zero, uses measured source text-minimum proportions when a first-row spanning/header table cannot satisfy all row text minimums inside the fixed graphic-frame height, measures spanning-cell text against its full `gridSpan` width for row reflow, including direct solid/gradient/pattern cell fills, direct cell borders, parsed `ppt/tableStyles.xml` fills/borders/text styling for common table style regions, solid and gradient table-background fills/effects spanning the full table background shape, row spans/vertical merges, cell text, round border joins, and double compound table border lines; remaining partial table features such as image/group cell fills, visible cell effects, non-flat line caps, bevel line joins, unknown compound-line variants, and unsupported line-end decorations are reported specifically
- diagram graphic frames can resolve related DrawingML diagram drawing parts, apply the package theme to their shape styles, paint their simple shapes/text inside the frame bounds, and report unsupported non-shape diagram content specifically
- lockedCanvas graphicData payloads lower supported GVML group child shapes and standalone `txSp` text shapes through existing static render primitives; full locked canvas and GVML host-drawing parity remain Partial
- chart graphic frames, OLE objects, controls, content parts, audio, video, and unknown graphic-frame payloads are detected from their source XML and relationships, preserved by the package writer, and reported during render; OLE/control preview pictures are still rendered through the normal picture path when present, charts remain Partial static-rendering work, and active controls, embedded application content, content parts, and media playback are not rendered
- text rendering preserves paragraph breaks, applies body insets including DrawingML defaults, supports simple left/center/right alignment, wraps words inside text bounds, honors `horzOverflow`/`vertOverflow` clipping versus overflow on each axis, expands unrotated text shapes with `a:spAutoFit` and widens no-wrap autofit boxes to contain measured text, applies explicit normal-autofit font scaling and applies authored line-spacing reductions only when normal autofit scales text below 100%, derives the largest fitting normal-autofit scale when `a:normAutofit` is requested while preserving authored break-line counts where scaling can avoid extra soft wraps, honors local `a:bodyPr` text anchoring and `anchorCtr` centering when present and otherwise inherits placeholder anchors, honors `spcFirstLastPara` for text-box anchoring, and uses first-pass DrawingML bullet prefixes and common alpha/arabic/Roman auto-number marker formats plus inherited paragraph margins, hanging indents, line spacing, fixed-point and percent before/after spacing, paragraph `fontAlgn` metric alignment, paragraph `rtl`/`eaLnBrk`/`latinLnBrk`/`hangingPunct` metadata preservation, bullet size, and paragraph `defRPr` font-family/font-size/color/bold defaults for body/content placeholders; unsupported text-body modes such as vertical text, body-level text rotation, vertical-overflow ellipsis, multi-column text, authored `rtl="1"` paragraph layout, picture bullets, and locale-specific numbering parity are reported specifically instead of using the former blanket simplified-layout report
- explicit `unsupported` JSON entries for visible objects that are detected but not yet painted
- empty placeholder shells are ignored when they have no visible content to paint

Full parity remains in progress. The real-world comparison harness is gated behind `PUPPT_RUN_REALWORLD_RENDER_TESTS=1` and compares all 61 fixture slides against the checked-in Apple Notes reference PNGs by default. Set `PUPPT_REALWORLD_REFERENCE_ROOT` to run a diagnostic comparison against a different checked-in reference set such as the secondary LibreOffice exports, and set `PUPPT_REALWORLD_ARTIFACT_DIR` to keep `got.png`, `reference.png`, and `diff.png` artifacts for failing slides.
